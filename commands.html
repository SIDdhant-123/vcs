Git Commands with the magic of story

  • git init
    • To initialise the git in our project. By using the command, git comes into action and starts tracking all our files in our folder, where we run this command. For all the new files we would see (U) symbol which tells that the file is untracked or if we see (M) symbol in the explorer window of vscode it says that the file is modified.
    NOTE: [Consider Git as a Director at backstage & actors of play(i.e natak) as files]-> so once we do "git init" command, the director(i.e git) knows that the actors(i.e. or files) are somewhere at the backstage but their exact location is not known to him(i.e. git) is called as untracked files(i.e. where exactly are the actors?)


  • git --version
    • It is used to check the git version installed in our system.

  • git config --global user.email "you@example.com"
    • It is used to set the user email id which is used as identification while commiting the code.

  • git config --global user.name "Your Name"
    • It is used to set the user name which is used as identification while commiting the code.
    NOTE: Run these two commands for setting email & username when you install Git for the very first time & for checking use "git config user.name" & "git config user.email".


  • git status
    • It is used to check the status of our code. For example it specifies whether our file is untracked, modified, staged or commited.
    NOTE: The Director(i.e. Git) ask his assistant to listdown (i.e. give status) of actors(i.e. files) whose location(i.e. untracked) we dont know at backstage .


  • git add [filename]
    • It is used to stage the untracked code file. Means the file from the working directory is moved to the staging area where this file is held for its changes to be commited to local repository.

  • git add .
    • It will add i.e. stage all the code files to the staging area.
    NOTE: The Director(i.e. Git) calls the actor(i.e files) one by one or on conference and brings them to backstage and now they are ready to go on stage to perform infront of the audience.


  • git commit -m "[message]"
    • It is used to commit the staged changes to the local repository with a -m option where the message is given to identify the recent changes.
    NOTE: commit is actually like savepoints in game and we will use this command whenever we sucessfully implement one of the important functionality of the code which means we have pushed that actor on stage infront of audience now.


  • git log
    • To check logs of all the comitts we made till now.
    NOTE: It is list of all the savepoints of the game(i.e. code here )we created till date & Type "q+Enter" to exit logs


  • git reset --hard [SHA number]
    • It is used to delete the recent commit change and go back to the previous commit working code represented by that particular commit's SHA.
    NOTE:[SHA number is the secured hash algorithm number which is unique for each commit.] if you have made a blunder in the latest code and you cannot find your error so you can go to previous commit & --hard means, you dont want any of the recent cod.So, take me to previous mentioned commit.


  • git reset --soft [SHA number]
    • It is used to go back to the previous commited code along with the latest code change still available that means you have to remove error yourself. The git log will only show previous commited logs but the recent commited log will not be there even though the change in this commit will be still available.

  • git remote add origin [url]
    • We are adding a remote origin means where our code should be pushed on cloud.

  • git branch -M main
    • We are creating branch for that repository called "main"

  • git push -u origin main
    • It is used to push the code from local to remote at the specified URLs origin's main branch.

  • git clone [url]
    • This will create clone on your collegues machine who are colaborating with you in your project.

  • git pull
    • Once cloned they need to only use pull command to get latest code on their machine.

  • git branch
    • It is used to check which are the branches and which is the current branch we are in now.

  • git branch [branchname]
    • It is used to create a branch with the mentioned branch name.

  • git checkout [branchname]
    • To switch from one branch to another branch.

  • git merge branchname
    • It is used to merge the new changes done in the mentioned branchname into the current branch.
